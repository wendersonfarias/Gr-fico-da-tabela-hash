Separate chaining
Hash collision resolved by separate chaining
In the method known as separate chaining, each bucket is independent, 
and has some sort of list of entries with the same index. The time for 
hash table operations is the time to find the bucket (which is constant) 
plus the time for the list operation.

In most implementations buckets will have few entries, if the hash function 
is working properly. Therefore, structures that are efficient in time and space 
for these cases are preferred. Structures that are efficient for a fairly large 
number of entries per bucket are not needed or desirable. If these cases happen 
often, the hashing function needs to be fixed.[10]

There are some implementations[11] which give excellent performance for both 
time and space, with the average number of elements per bucket ranging between 5 and 100.
